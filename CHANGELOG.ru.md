# История изменений

Все значимые изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)
и следует принципам семантического версионирования.

---

## [Unreleased]

### Добавлено
- Переиспользуемые шаблоны команд openHASP (`OPENHASP_TEMPLATES`)
- Возможность использовать один шаблон для нескольких устройств
- Привязка устройств к шаблонам через `OPENHASP_DEVICES`
- Опциональный отладочный лог загрузки `.env`
- **Автоматическая регулировка яркости** по времени восхода/заката через библиотеку `suntime`
- Три режима яркости: дневной, сумеречный и ночной
- Настраиваемые смещения восхода/заката (`SUNRISE_OFFSET_HOURS`, `SUNSET_OFFSET_HOURS`)
- Настраиваемые уровни яркости (`BRIGHTNESS_DAY`, `BRIGHTNESS_NIGHT`, `BRIGHTNESS_TWILIGHT`)
- Расчёт положения солнца по координатам (`LATITUDE`, `LONGITUDE`, `TIMEZONE`)
- Опция игнорирования подтверждённых проблем (`IGNORE_ACKNOWLEDGED`)

### Изменено
- Рефакторинг логики формирования MQTT-команд для openHASP
- Разделение идентификаторов устройств и сценариев поведения
- Уменьшено дублирование кода между openHASP-устройствами
- Конфигурация объединена в один скрипт
- Расширен отладочный вывод: время восхода/заката и расчёт яркости

### Исправлено
- Проблемы совместимости с версиями Python ниже 3.10
- Ошибки валидации параметров Zabbix API
- Избыточное количество вызовов Zabbix API

### Зависимости
- Добавлено: `suntime` для расчёта восхода/заката

---

## [0.1.0] – Начальная версия

### Добавлено
- Интеграция с Zabbix 7.x через API-токен
- Получение активных проблем и определение максимального severity
- Сопоставление severity → цвет (green / yellow / red)
- Отправка MQTT-команд на openHASP устройства
- Поддержка нескольких openHASP экранов
- Загрузка конфигурации из `.env`
